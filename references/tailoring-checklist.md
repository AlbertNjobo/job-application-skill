# Resume Tailoring Checklist

## Before Submitting Any Application

### ATS Compatibility
- [ ] Exact skill keywords from JD appear in resume
- [ ] Standard section headers (Experience, Education, Skills)
- [ ] No tables, graphics, or columns
- [ ] File is PDF or DOCX
- [ ] Font size 10pt+ for body text

### Keyword Integration
- [ ] Top 5 JD requirements appear in top 5 skills
- [ ] Keywords placed in: Summary, Skills, Experience bullets
- [ ] Keywords used naturally (not stuffed)
- [ ] Exact JD phrasing used where accurate

### Content Tailoring
- [ ] Summary mentions the exact job title/function
- [ ] Most relevant experience positioned first
- [ ] Each job's lead bullet addresses JD's key requirement
- [ ] Irrelevant experience de-emphasized or removed
- [ ] Page count: 2 for technical, 1 for admin/data entry

### Truth vs. Tailoring Line

**Acceptable (highlighting):**
- Reordering true information
- Emphasizing relevant experience
- Using industry-standard terminology
- Adding context to vague statements
- Matching language to job description

**Unacceptable (lying):**
- Adding skills you don't have
- Changing numbers or metrics
- Creating fake experiences
- Claiming titles you didn't hold
- Stating certifications you don't have

### Quality Checks
- [ ] Zero em dashes in document
- [ ] No personal pronouns in experience bullets
- [ ] Action verbs start every bullet
- [ ] At least 1 metric per bullet where possible
- [ ] No typos or grammatical errors

### Final Verification
- [ ] Run `validate-output.sh <slug> <company>`
- [ ] Cover letter is exactly 1 page
- [ ] CV is correct page count
- [ ] soul.md portfolio registry updated
- [ ] File named: Lawrence_Njobo_<Role>_<Company>.pdf
